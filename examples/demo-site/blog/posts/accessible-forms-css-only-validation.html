Accessible Forms: CSS-Only Validation

Published: 2025-02-07 ¬∑ a11y forms

On this page: Labels and Hints ¬∑ Valid and Invalid ¬∑ Error Messaging

Modern selectors like :user-valid and :user-invalid let you provide helpful, accessible validation states with no JavaScript.

Accessibility is a product feature ‚Äî not an afterthought.

Labels and Hints

Always associate labels, indicate required fields, and provide concise instructions.

Valid and Invalid

Use color and subtle backgrounds to reinforce state, and pair with programmatic cues.

Tip: Ensure sufficient contrast for all states in light and dark themes.

Error Messaging

Show helpful messages near fields and ensure they‚Äôre announced by assistive tech.

CSS Example

@layer components {
  form-field:has(input:user-invalid) field-error { display: block; }
  input:user-invalid { border-color: var(--color-error); }
  input:user-valid { border-color: var(--color-success); }
}

Internationalization

Keep labels concise and avoid culture-specific assumptions. Validate format rather than language-specific content where possible, and ensure date/number formats are localized when you add server logic.

Testing Checklist

  • Keyboard-only completion with visible focus states.
  • Screen-reader announcement of labels, errors, and help text.
  • Contrast ratios for text and indicators in all themes.
