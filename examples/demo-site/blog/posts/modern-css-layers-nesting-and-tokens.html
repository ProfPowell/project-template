Modern CSS: Layers, Nesting, and Tokens

Published: 2025-02-03 ¬∑ css tooling

On this page: Layers ¬∑ Tokens ¬∑ Nesting

CSS layers help control cascade order, tokens centralize design decisions, and native nesting simplifies selectors ‚Äî together they make styles more maintainable.

Treat your CSS like an API: stable tokens, clear layering, clean scopes.

Layers

Separate reset, tokens, layout, components, and responsive rules to manage specificity and intent.

Tokens

Define design tokens for color, spacing, and typography. Reference tokens everywhere to reduce drift.

Nesting

Native nesting keeps selectors compact and local to components, improving readability.

Examples

/* Tokens */
@layer tokens {
  :root { --brand: #2563eb; --surface: #fff; }
}

/* Components */
@layer components {
  .card {
    background: var(--surface);
    border: 1px solid #e5e7eb;
    & h3 { color: var(--brand); }
    & a { text-decoration: none; }
  }
}

Migration Tips

  • Introduce layers gradually: start by wrapping existing CSS in a single @layer and split later.
  • Centralize colors and spacing as tokens; refactor components to consume them.
  • Use a linter/formatter to keep nesting shallow and intentional.

Browser Support

Modern browsers support layers, nesting, and custom properties. For older browsers, the cascade gracefully ignores unknown at-rules; keep specificity modest and provide sensible defaults.
