The View Transitions spec lets documents animate between states and pages, enabling shared-element effects with minimal markup and no JavaScript.

Continuity matters: animate what the user mentally tracks, not everything.

Naming Shared Elements

Assign the same view-transition-name on both pages to link elements across navigation.

Timing Considerations

Note: Ensure images decode before navigation so they can be captured for the transition.

Use eager decoding for heroes and keep DOM stable during navigation.

Styling The Transition

You can target the pseudo-elements to adjust duration and easing for specific shared elements:

/* Customize a named transition */
::view-transition-group(card-1) { animation-duration: 250ms; }
::view-transition-old(card-1),
::view-transition-new(card-1) { animation-timing-function: ease-out; }

Limitations And Fallbacks

  • Cross-document transitions require same-origin navigation and paint-ready elements.
  • Elements must be visible to be captured; ensure the target is scrolled into view on return.
  • Respect prefers-reduced-motion: allow the system to disable animations.
