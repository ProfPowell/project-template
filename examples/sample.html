About This Approach

We combine HTML5 semantic elements with XHTML-strict syntax rules for clean, valid markup.

Key Principles

  • All void elements use self-closing syntax: <br/>
  • All tags are lowercase
  • All attributes are double-quoted
  • All elements are explicitly closed

Semantic Elements

Use semantic HTML5 elements instead of generic <div> elements:

  • <header> for page/section headers
  • <nav> for navigation
  • <main> for primary content
  • <article> for self-contained content
  • <section> for thematic grouping
  • <aside> for tangential content
  • <footer> for page/section footers

Quick Reference

Common void elements with self-closing syntax:

  • <br/>
  • <hr/>
  • <img src="" alt=""/>
  • <input type=""/>
  • <meta/>
  • <link/>
